Accman Refresh : ARF01 - Complexity Matrix

Created by Unknown User (ujs079), last modified on Feb 02, 2015

Terms

The matrix table depicts "touch points" at code level which can be used to determine complexity. The header sections define a broad area with sub sections showing more specific elements. Each row in the table represents a translated script with its associated JIRA issue number. 

Storage
This section identifies storage elements usage by a script where CRUD (Create, Read Update, Delete) operations (represented by a character code defined in the legend) are added to each subset, where the subset represent the actual storage mechanism. It can be used to represent database, file based and directory (AD) level structures. 

Lib
In both Perl and .NET shared code is used. The libs code has been ported to include processing business rules is separate functional units.  In general these shared libraries contain routines capable of database interaction and transaction generation. The use of a Libs are taken in to account when determining the size (See Metric).

Metric
This section defines alternate attributes for identifying complexity. Currently it shows child transactions created by the script.  Conditional logic should not be taken into account when populating this value. The script size is denoted by a character code defined in the legend. The line count method is not used as comments and logging can give a misleading picture of the true size of the script especially when considering scripts can reference Lib code.
