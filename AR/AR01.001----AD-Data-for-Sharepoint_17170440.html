Supporting Context

The contact details in the Online Staff Directory and staff reporting lines should be held on Sharepoint. The following fields have been identified for possible inclusion in the AD interface to Sharepoint. The MoSCoW ratings have been attributed following a discussion with Unknown User (uis864), but need to be verified.

  • User Name (ID) [M]
  • First Name [M]
  • Last Name [M]
  • Job Title [S]
  • Manager (of Team) [S]
  • Team (to which the User belongs) [S]
  • Unit Name (Department School etc) [S]
  • Phone Number [M]
  • Email Address [M]
  • Building Location [C]
  • Floor/Room Number [C]

Currently AD holds the following information:

  • User ID
  • First Names (all first names in the same field, separated by spaces)
  • Initials (Of all first names)
  • Last Name
  • Email Address

There are issues with the location of data in incorrect fields (eg The Job Title is used to identify Staff employees, not a specific job). Initial contact for further information Mick Stephenson (or any of the AD team on the 5th floor)

The Staff Directory is manually updated by nominated administrators in each School or Department. Not all input fields are completed when a new starter is entered, or amendments are made. The Staff directory does not hold the User ID, this will be an issue when identifying a unique key for migration or transfer of this data. I. The fields that are completed are:

  • First Name
  • Surname
  • Job Title
  • Unit Name
  • Team
  • Phone Number
  • Email Address
  • Building Location
  • Room Number
  • Category of Office

 

Reporting lines are not used extensively within Sharepoint. ITS populated the AD fields as a one off exercise, but this has not been updated and has not been undertaken by any other school or division.

The majority of Staff information can be taken from SAP. The SAP system holds up to 6 levels of organisational structure, so the extract will need to select the correct level for each staff member. Initial SAP contact - Shamshudin Kapasi


The contact for the Associate database is initially Mark Winiewski
