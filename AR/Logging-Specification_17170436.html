Accman Refresh : Logging Specification

Created by Unknown User (ais008), last modified on Jul 04, 2014

In an ideal world, what's the perfect logging system?

  • not the present system! 
  • Try to replicate features of the unix syslog daemon?
    • multiple levels of reporting - ERROR, WARNING, INFO, DEBUG
    • log errors on a remote server - i.e. processing occurs on the accman back-end but logs are written both locally on the accman back-end itself and on a central server where they can be read without having to log on to the accman back-end
    • using syslog conventions would allow teh use of standard tools to view and analze log files
    • ability to change the logging level 'live' without having to stop and restart any processes (and hence not requiring an RFC!)
  • automatic log rotation and houskeeping
  • on accman back-end, write all log files to D: drive rather than a mixture of C: drive and D: drive as at present
  • for each transaction logged, show date/time and the username (or other object) that it relates to
  • show a clear return code at the end of the log file segment for a transaction
  • we currently have both individual per-transaction log files and 'cumulative' log files
    • re-examine to achieve greater consistency
