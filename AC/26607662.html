Repeated Code to send exception E-Mails

The AccMan web application's web.config is configured as followsÂ <add key="ErrorEmailRecipient" value="its-development-team@salford.ac.uk" />

At this time, there is repeated code use of SmtpClient class being called from Global.asax files on ApplicationError across the web applications in the AccMan solution

These projects are

  • AccMan
  • AccmanSelfActFE
  • AssociateRequest
  • AccManWebService
  • StaffSelfActivation

Error logging In-line code within aspx page markup

It is advised that the error logging logic within the page AccMan.NET\AccMan\Views\Shared\Error.aspx be refactored into a common error logging call.

Action Points

  1. Agree on a common standard exception logging format e.g. ErrorId, DateTime, ErrorType, Message, SystemOriginated, ProcessName, User.
  2. It is advised to review the e-mails sent to the above ErrorEmailRecipient account and determine the current requirement of E-Mail logging level.
  3. This could be refactored into a common Email sending component. I would suggest the E-Mail component throw it's own meaningful exceptions for the calling assembly to catch and log or handle otherwise.
    Currently the component Salford.AccMan.BackEnd.Processes.EmailSender exists and may be used instead of the duplicated code in Global.asax.cs files.
  4. Relocate the error handling of the Error.aspx page in AccMan to common logic and format the exception in a standard format.
