Details

There is a manual process used to delete AD accounts and mark Accman accounts as deleted. This should be automated.

Note the current logic doesn't handle students that are absent or UG accounts that got replaced by PG accounts. Note replacement PG accounts were stopped being generated in 2012.

There are a number of Left/Disabled accounts that are not being included in the logic. See the spreadsheet V:\ITS\Programme Management\Applications\Support\Applications\Accman v2.0\Undeleted Unused AD Accounts Research.xlsx.

See V:\ITS\Programme Management\Applications\Support\Applications\Accman v2.0\Usage\Accman v2.0 - USG - Processing AD Account Deletions.docx.

 

Comments:

I have looked into this issue and found following:

  1. If you pass 'disable_date_range_start' value to somewhere in 2011 will fix most of the account. For example pass '2011-01-01' as 'disable_date_range_start' value and run all three powershell script. 
  2. There are many student account which have account status as 'Disabled' but Person status set to either 'Present' or 'Absent'. However if you check the disabled reason, it says 'Now using xyz123'. This means this person is now using different account. I have amended the query which will bring all these account

However there is one more BIg group which account status is 'Disabled' but person status is 'Absent'. Now I am not sure what to do with these accounts. Only thing I found is there are couple of account of these kind have disabled reason as 'Student has left'

I suggest to first execute first two points and investigate absent account later.
