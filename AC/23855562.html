Details

  • Following an initial investigation the original purpose / definition of the Accman person state of 'missing' was:

    • The student has disappeared from SIS imports, but has never been shown as having left; or the student had an account in the pre-AccMan (userreg) database, but has never been imported as a current student. Staff: record  has disappeared from the SAP import.

  • With the above in mind can we investigate how Accman sets the 'missing' state and how it would process someone who became missing
