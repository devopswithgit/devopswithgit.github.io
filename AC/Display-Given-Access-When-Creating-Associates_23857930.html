Accman : Display Given Access When Creating Associates

Created by Paul Daniels on Feb 01, 2016
SummaryCreating Associates doesn't show what access is provided.
Issue TypeEnhancement
PriorityHigh
Raised ByPaul Daniels
Date Raised01 Feb 2016 
SourcePaul Daniels 
JIRA Issue(s)

 

Details

When an associate is created, there is no indication as to what will be given by the selected class so it can be confirmed by the sponsor/creator. The main issue is classes that don't provide exchange accounts displaying an exchange email address.

I believe is would reduce cases if the Create Associate page also displayed the access the selected class will give. It would also mean selecting the class first.

For classes that don't provide an Exchange account, it should say something like "None Given" next to the "Email : " prompt. Should also be included on any emails to the sponsor.

For classes that don't provide Library access, it should say something like "Not Available" next to the "Library Site : " prompt. Should also be included on any emails to the sponsor.
