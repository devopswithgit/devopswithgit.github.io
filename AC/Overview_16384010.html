Accman : Overview

Created by Unknown User (uis894), last modified on Sep 10, 2019

Introduction

Accman (short for Account Manager) is the university's system for managing user accounts. It takes feeds from other systems such as Banner and SAP, and ensures that the users' accounts are correctly provisioned. Once an account is created in Accman, it then pushes the user's details into a variety of other systems such as Janus (for managing ID cards) and CMIS (for managing student timetable data). Accman also handles the full lifecycle of an account, updating user details when necessary and ultimately disabling or deleting accounts when a user leaves the university. 

Accman has evolved over a number of years, starting life as a collection of Perl scripts, with more recent additions being made in .NET. 

Accman has never been properly documented, so this is an attempt at doing just that. 

System Overview

New accounts are fed into Accman by Banner (students) or SAP (staff). Accman then provides their details to numerous other systems within the university.

Accman Eco System

Accman itself consists of several different applications hosted on five different servers, plus one database server. Note, this diagram only shows the Accman components themselves. Each application may also communicate with one or more external systems (and their databases) too.
