Accman : SWR0467 LoXS Interface

Created by Unknown User (uis881), last modified by Unknown User (uis894) on Sep 04, 2012

Introduction

The initial specifications for the LoXS interface turned out to be incorrect.  As such, it transpires that while additions can be made to the system, changes and deletions cannot.

As such, there is a need to change the LoXS interface to allow for these changes.

In real terms, this means altering the LoXS DAO object such that:

  • if no card number is supplied, it is assumed that the action is disable
  • in this case, the card number is retrieved from the LoXS database (SELECT Kaartid FROM Gebruiker WHERE Sleutelveld = @rollnumber)
  • the card status is set to 0 for an active card and 1 for an inactive card (note that this is logically inconsistent)

To test, create a type 107 transaction as follows:

  • New User: ROLLNUMBER parameter = rollnumber and CARDNUMBER parameter = 56-bit card number
  • Update User: ROLLNUMBER parameter = rollnumber and CARDNUMBER parameter = new 56-bit card number
  • Disable User: ROLLNUMBER parameter = rollnumber and no/blank CARDNUMBER parameter

Note that, as there is no LoXS test system, testing has been successfully performed against the live system using a single known user (rollnumber 304476, 56-bit number 00036070408583472132) who was disabled in Janus.
