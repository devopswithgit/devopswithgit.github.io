Accman : Reporting of transaction Processing Failures

Created by Paul Daniels, last modified by Stephen Harrison on Oct 30, 2015
SummaryReporting of transaction Processing Failures
Issue TypeBug
PriorityHigh
Raised ByPaul Daniels
Date Raised21 Apr 2015 
SourcePaul Daniels 
JIRA Issue(s) ARF-216 Open

 

Details

Currently there is no way of knowing if a transaction has failed except for checking logs on the Accman BE. Would it be possible for the transaction table to be updated with a failure message?

This is particularily useful for the password resets, both AD and Office 365. See case 145569 which was due to a failed O365 password change. There does not appear to be any logging that says why it failed. More cases are being raised around password resets due to the new rules being implemented and no immediate feedback on the success when the backend processes the transaction.
