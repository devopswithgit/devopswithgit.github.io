Accman : ScriptPathChange - AD Modification

Created by Unknown User (uis894), last modified on Apr 25, 2013

Introduction

Issue AC-28 is to set a value for the 'scriptPath' attribute on an AD user account when it's created.

The value is set when a new user is created in AD (via a type 93 (ActiveDirectoryNewUser) transaction). It has never been set before, but we are now setting it to "UoS-Login.vbs".

It is set using this line of code, which is in the class 'ActveDirectoryBaseProcessor.cs".

Line from ActiveDirectoryBaseProcessor.cs
userDe.Properties["scriptPath"].Value = parameters["scriptPath"];

It is configured in the quartz_jobs.xml file in case it ever needs to change, as follows:

Quartz Job Configuration (Transaction Type 93)
<entry>

    <key>ScriptPath</key>

    <value>UoS-Login.vbs</value>

</entry>

The code is all on the following branch in Subversion:

svn://isd-sysarch01.isdads.salford.ac.uk/UoS_IdM/Accman.NET/branches/ScriptPathChange
