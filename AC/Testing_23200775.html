Created by Julia Kelbrick, last modified on Oct 09, 2015

This section is concerned with describing procedures for testing AccMan and includes details of the configuration steps required to support this.

1. Testing of the AccMan Rewrite 2015

Following the rewrite of the AccMan Perl scripts to .Net, two test processes were developed to ensure that the processing results of the .Net application is the same as that of the Perl scripts being replaced.  The following sections describe how to reproduce these tests.

1.1  The end-to-end testing of Perl Scripts compared to .Net processes excluding the Banner Import stage.

Summary:  This stage of testing utilised two new temporary copies of the AccMan database, copied from the production database.  One database (MailPerl) was used to run the Perl Scripts against and the second database (MailNet) was used by the new .Net application.  6350 test cases were inserted in to the Banner_Import tables of each of the databases.  The result of the processing was to create and update records in the AccMan Person table.  At the end of testing, the MailPerl.Person and the MailNet.Person tables were compared in order to identify any differences in processing between the new .Net application and the Perl scripts being replaced.

1.2  The end-to-end testing of Perl Scripts compared to .Net processes including the Banner Import stage.

Summary:  In this stage of testing, production data copied to the MailPerl and MailNet databases was used as the basis for testing and the "Banner Import" process was added to the the test cycle in order to end-to-end test the processing of data from the start of the process when it is imported from Banner, right through to record creation and update of the Person table.
