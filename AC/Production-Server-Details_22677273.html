Introduction

This page is a crib sheet for anyone who needs to support the Accman server, particularly in the event of a problem.

Server details

Server name: uos-p-acbe-01.isdads.salford.ac.uk

Database name: mail, on accman.salford.ac.uk. This is a MySQL database listeing on port 3306.

Connecting to the server via Remote Desktop

You need to log in to the server using the service account svc-prd-accman. You can get the password from the development team.

You cannot log in directly to the server. You must first remote desktop to the Management server, and from there remote desktop to the Accman server. 

You must use the /admin switch when connecting via remote desktop. Otherwise when you log out, the scripts will all stop running!

  1. Remote desktop to the management server by entering mstsc /admin from the Start menu, and entering uos-p-adma-01 for the Computer name.
  2. Enter your own network username and password.
  3. When on the management server, remote desktop again (run mstsc /admin from the management server, not from your own PC) to uos-p-acbe-01.mgt.isdads.salford.ac.uk.
  4. You may need to accept a few warning messages about certificates. 

The Desktop

You will only be able to see the running scripts if you log in using the svc-prd-accman service account. You must always use this account when logging in.

Accman Processes

The Accman system is comprised of several different components. Accman can be considered to be 'up and running' if the following are all running

ComponentPurposeNotes
ConsoleSchedulerService (Main)

This is what runs the various Quartz jobs which are responsible for processing Accman transactions.

You can start it using the desktop shortcut called: ConsoleSchedulerService

To stop it, open the command prompt, and type exit.

This is the main component of Accman. it's what creates AD accounts, processes password changes, updates Janus security details etc.
ConsoleSchedulerService for Scripts

This is another Command Prompt based application responsible for running Quartz jobs related to the SalSync process.

You can start it using the desktop shortcut called: ConsoleSchedulerService for Scripts

To stop it, open the command prompt, and type exit.

The SalSync is affectionately knows as the 'Mike Hall' script. It does some Exchange-related things when a new account is set up.
ConsoleSchedulerService for SAP

This is another Command Prompt based application responsible for SAP running Quartz jobs related to the SAP process.

You can start it using the desktop shortcut called: ConsoleSchedulerService for SAP

To stop it, open the command prompt, and type exit.

 
ConsoleSchedulerService for ALMA

This is another Command Prompt based application responsible for running Quartz jobs related to the ALMA process.

You can start it using the desktop shortcut called: ConsoleSchedulerService for ALMA

To stop it, open the command prompt, and type exit.

 
PowerShell Scripts

There are four PowerShell scripts, responsible for creating Office365 accounts

To start: click on the Start PowerShell Scripts desktop shortcut

To stop: click on the Stop PowerShell Scripts desktop shortcut

There is one PS script each for the following tasks:

  • Create Office365 account
  • Change Office365 password
  • Enable Office365 account
  • Disable Office365 account

 

Log Files

All Accman Logs are now written to D:\AccmanLogs, with the exception of the CMISLecturerImport process, which still logs to C:\Logs.

The SchedulerService.log is rotated so should never fill up the disk. 

D:\AccmanLogs\TransactionLogs contains one log file per transaction processed and can therefore this folder can become very large in size.

Troubleshooting

Low disk space

If the server is running low on disk space, any log files can be moved off the server to free up space. 

Please zip up log files before deleting them, in case the information they contain are needed later. The files compress well (several GB of log files will compress to just a few hundred MB when zipped up)
