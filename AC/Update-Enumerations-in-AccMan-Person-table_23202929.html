Accman : Update Enumerations in AccMan Person table

Created by Julia Kelbrick, last modified by Jon Besson on Dec 04, 2015
SummaryValidation on Accman Person tables need updating
Issue TypeBug
PriorityMedium
Raised By

Julia Kelbrick

Date Raised11/11/2015
SourceJulia
JIRA Issue(s) ARF-296 Closed

 

Details

Summary:

The AccMan.Mail database uses Enumerations to validate the values being changed in various columns in the Person table.  Over time, some of the codes being used in Banner have become out of synch with these enums in Accman.  Consequently, the BannerForImport process is unable to update these values, resulting in it attempting to do an update each time it runs and creating a corresponding transaction type 97 (UPDATE_JANUS_USER_DETAILS).

Details:

The following two SQL statements need to be run against the Production Accman.Mail databases and all of the test Accman databases in order to make these data updates work in future and to reduce the creation of type 97 transactions.

ALTER TABLE mailnet.person CHANGE stu_styp stu_styp enum('C','E','N','R','T','D','F','0','Y','Z','S');                          To add codes 0, Y, Z, S

ALTER TABLE mailnet.person CHANGE stu_stst stu_stst enum('AS','IS','DE','IG','RR','SU','WD','WU','IN','WN', 'EA','XP');                   To add codes EA and XP
