Accman : Difference in Column Widths between AccMan and Banner causing BannerForImport Problems

Created by Julia Kelbrick, last modified by Jon Besson on Dec 04, 2015
SummaryDifference in Column Widths between AccMan and Banner creating BannerForImport Problems
 
Issue TypeBug
PriorityMedium
Raised By@Julia
Date Raised11/11/2015
Source@PaulDaniels
JIRA Issue(s) ARF-295 Resolved

 

Details

There are differences in column widths between AccMan and Banner that result in the data being truncated in the AccMan Person table following the BannerForImport process.  Consequently, when BannerForImport is run on future occassions it is unable to match the two values as equal and attempts to update AccMan each time and in addition creates a corresponding type 97 transaction (UPDATE_JANUS_USER_DETAILS).

To resolve this, the code in the "StudentActions" class which determines if the Banner and AccMan values are different should be changed to do the comparison based on the AccMan column width.
