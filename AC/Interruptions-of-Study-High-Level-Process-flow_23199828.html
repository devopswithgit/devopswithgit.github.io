Accman : Interruptions of Study High Level Process flow

Created by Unknown User (ujs077), last modified on Jul 29, 2015

Summary

Student request Interruptions of Study which then approve or reject by different body in the University. If approved their status will change to Absent and their account will be disabled.

 

  • Student Request Interruptions of Study
  • Programme administrator/Associate Head of school/Associate Dean/Member of Fees and Award team Approve or Reject the request
  • If approved SA (originally SID) update the banner and changed student registration status from ‘RE’ to ‘IN’
  • AccMan Transaction ‘Banner4Person’ and/or ‘Assign’ read these student data and add/update person table
    • Banner sent enrolment code (SFBETRM_ESTS_CODE) which AccMan map to Person Status (using student_ests table)
    • Banner ‘IN’ code translated as ‘Absent’ in AccMan
  • AccMan transaction ‘StudentAutoState’ Read person data with status ‘Absent’ and mark them disable
  • Disable account transaction then generate csv file for this data
  • Active Directory Disable user transaction then read this csv file and disable account
