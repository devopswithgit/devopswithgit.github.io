Accman : RFC1567: LDAP Removal

Created by Unknown User (uis881), last modified by Unknown User (uis894) on Sep 04, 2012

Introduction

This RFC is about removing the LDAP servers (i.e. Novell) and making the appropriate changes to AccMan in response.

In this particular case, there is an issue to do with the POSIX account UIDNumbers that AccMan pushes downstream for the computer science students who use a ubuntu dual-boot set-up and require the UIDNumber to be present as an attribute in order for this to work.

The required changes are:

  • add posix account id to Account object
  • update account DAO to retrieve posix account id
  • amend create new AD account back end process to add posix account id value into AD

The code change writes the UIDNumber generated as part of the resgister.pl script to AD during AD account creation.

To test, check that the UIDNumber attribute has been added to a newly created AD account, and that the value matches that allocated to the account in the posixaccount database table in AccMan.
