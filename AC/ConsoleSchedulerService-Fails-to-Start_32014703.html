Accman : ConsoleSchedulerService Fails to Start

Created by Unknown User (uis894), last modified on Feb 06, 2017

When deploying some of the ConsoleSchedulerServices from Bamboo and then trying to start them, this error can occur

The application has failed to start because its side-by-side configuration is incorrect. Please see the application event log or use the command-line sxstrace.exe tool for more detail.

This is caused by ConsoleSchedulerService.exe.config being saved in the incorrect codepage (usually UCS-2 Little Endian), and is a problem that we have yet to solve. 

The fix is easy: open the file in Notepad++ and go to the Encoding menu. Select encoding UTF-8 and save the file. It should now start. 

For some reason, the file when zipped up into the artifact is being set with an unusual codepage that Windows doesn't seem to like.
