Accman : Send_Mail transaction doesn't retry on failure

Created by Paul Daniels, last modified on Apr 04, 2016
SummarySend_Mail transaction is not retried after failures
Issue TypeBug
PriorityMedium
Raised ByPaul Daniels
Date Raised04 Apr 2016 
SourcePaul Daniels
JIRA Issue(s) ARF-346 - Send_Mail transaction is not retried after failures Closed

 

Details

If the Send_Mail transaction fails for any reason, it should retry sending it after given amount of time. Failures can be caused by Accman running out of memory or Exchange being to busy. This can cause cases to be raised after an associate account is raised, as the credential details are not sent to the sponsor. Case 189056 lists three associate accounts that did not have an email sent. Changing the transaction state from pending to waiting manually is the manual work around.
