Accman : Testing and Mocking Framework

Created by Unknown User (aaa45816), last modified on Jun 15, 2016

Testing

The projects current use Visual Studio Test Tools. The developer may choose to utilise NUnit and it's own front-end.

Mocking

The developer may wish to use either Moq, Rhino or similar. The current framework used is FakeItEasy, for it's simplicity in setup and configuration of mocks.

FakeItEasy

FakeItEasy is used in the test projects under the Test solution folder. The developer may install FakeItEasy from the NuGet package manager as follows:

  1. Click 'Project' menu
  2. Select 'Manage NuGet packages'
  3. (VS2010) On the left, select 'online' and search for 'FakeItEasy'
  4. In the search results click 'Install' on the 'FakeItEasy' result with Package ID: 'FakeItEasy' (Created By Patrik Hagne)

Some points to note for FakeItEasy

  1. Only public or virtual methods can be faked e.g. public sealed class MyClass { ... } cannot be faked as MyClass will not be exposed.
  2. If an internally called method of a faked object requires arguments, but the arguments are not important to the test, the .WithAnyArguments configuration can be called
    e.g.

    A.CallTo(() => MyClass.MethodWithLostOfParams("","",null)).WithAnyArguments().DoesNothing();

    Note, some dummy arguments still have to be defined in the A.CallTo config as above, to obey the Faked method signature but when actually called, the step will ignore them.
