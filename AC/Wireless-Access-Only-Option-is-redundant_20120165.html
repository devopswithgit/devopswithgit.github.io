Accman : Wireless Access Only Option is redundant

Details

The option below, which is displayed on the 'Create New Associate' screen in the Accman Front End is probably redundant so should be removed to prevent users from thinking that it actually does something.

It used to end up generating a 'Restrict Network Access' transaction back in the day when we used NDS instead of AD. However this transaction (type 86) is commented out on the production server, so has no effect. It used to call some JRBUtils functions.

From an email I just sent to Lee Wilcox regarding this option:

Email

I’ve traced through the code and it looks as though that ‘Wireless Only’ option is obsolete now. It used to do something with NDS that let you specify wireless only, but that’s since been disabled. It should probably be removed from the Accman Front end as the users probably think it still does something.

Checking with Tom, you either have an account in AD or you don’t, and wireless access is just one of the perks of having an AD account nowadays. There’s no group (for example) that you could be added to, that says ‘this person only has wireless access’.
