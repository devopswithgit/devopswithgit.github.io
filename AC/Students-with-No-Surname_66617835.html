Accman : Students with No Surname

Created by Unknown User (uis894) on May 13, 2019

Introduction

Some students have no surname. This can cause problems when creating Active Directory accounts because AD needs to have a value.

This can result in the failed AD Transaction (type 93) getting stuck in aÂ pending status. The errors in the log files resulting from this scenario look like this:

   at System.DirectoryServices.AccountManagement.Principal.Save()

   at Salford.AccMan.BackEnd.Processes.Processes.ActiveDirectoryBaseProcessor.saveUser(ADUserClassExtensions userExt) in d:\bamboo-home\xml-data\build-dir\AC-AP151-JOB1\Accman.NET\AccManBackEndLibrary\Processes\ActiveDirectoryBaseProcessor.cs:line 1808

Solution

The solution is to enter a space character for the student's surname. This can be done by updating the RealName column in the Person table to '<space>', using the following SQL, and then re-run the failed transaction by setting the status back to waiting.

-- Update the surname to a space
update person set realname = ' ' where personid = <personid> limit 1;

-- Re-run the transaction
update transactions set tr_state = 'waiting' where trid = <transaction_id> limit 1;
