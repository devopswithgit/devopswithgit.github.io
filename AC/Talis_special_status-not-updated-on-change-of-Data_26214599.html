JIRA Issue(s) ARF-350 - Talis_special_status not updated on change of Data Open

 

Details

If a student has been granted a "Extended Borrower Status" via the Accman frontend, it is not being updated when the referenced data is changed. A manual Revoke/Grant process via the Accman frontend is required to update talis_special_status.special_status.

See user EPM315 as an example. Was ELB but became ELPG after manual workaround.

Note this isn't handled by a transaction so no audibility

This affects the Alma User Group calculation.
