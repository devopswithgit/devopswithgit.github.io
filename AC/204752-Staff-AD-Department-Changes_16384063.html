Accman : 204752 Staff AD Department Changes

Created by Unknown User (uis881), last modified by Unknown User (uis894) on Sep 04, 2012

Staff changes on the SAP system to not propagate departmental changes to the AD system.

This fix is required to address this issue.

This fix requires minor changes to sap_import.pl:

sub updateStaff {

...

my $badpernr = 0;
$reject = 0;

my $dept_change = 0;

then change:

logchange ('dept_code') if $db_dept_code ne $dept_code;

to:

if ($db_dept_code ne $dept_code)
{
logchange ('dept_code');
$dept_change = 1;
}

and finally add:

if ($dept_change)
{
return 0 unless $childTrID = generateTransaction ($dbh, 121, $db_personID, %transactionParams);
}

What these changes do is create a flag that gets set if the department code changes, and then creates a type 121 transaction (which updates the AD department field) if that flag is set.

Testing

To test, change the department code in the sap_import table, run the sap_import.pl script, and see if a type 121 transaction is generated. Â The type 121 transaction is already live and working, so no testing of this element is required.

Rollback

To rollback, simply run the previous version of the sap_import.pl script instead of the newly modified version.
