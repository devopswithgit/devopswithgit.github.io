Accman : Improved Logging and Log Rotation

Created by Paul Daniels, last modified by Stephen Harrison on Oct 20, 2015

 

 

SummaryImproved Logging and Log Rotation 
Issue TypeEnhancement
PriorityHigh
Raised ByPaul Daniels
Date Raised20 Apr 2015 
SourcePaul Daniels 
JIRA Issue(s) ARF-194 Open

 

Details

To improve tracking of transaction failures it would be very helpful if logs had a defined format, including date, time, script name, identifier, descriptive error message.

There should also be a system in place to rotate and compress old logs so that the BE doesn't run out of diskspace. There is software readily available to handle this, e.g. logrotatewin.

Comments:

In terms of the log rotation, we need to stick to the existing ITS policy of only logging for the last 90 days.
