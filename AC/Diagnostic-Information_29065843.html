JIRA Issue(s) ARF-364 - Diagnostics for Accounts and External Systems Closed

Summary

Modify Accman so that it is easy to see some diagnostic information to assist in solving some issues that occur regularly. There are two main types of information to display.

  1. Information relating to an individual account/person e.g. the date they last appeared on a Banner/SAP import
  2. System-wide information such as whether Accman is having difficulty connecting to an external system e.g. Janus

Acocunt/Person Related Diagnostics

This should be another tab on the account screen and can display a variety of information.


Suggest creating a new table for storing this kind of information rather than adding extra columns e.g. to the Person table. The table can be generic so that it can hold lots of different types of details

TypeValueIdentifierDescription
DateLastOnBannerImport01 Jun 2016 12345Date a student was last imported. Identifier is PersonID
DateBecameAlumnus09 Jun 2016 98765Date a student last became an alumnus

The tab can also be used to show other information i.e. not just from the above table. For example, it could display their current status in Active Directory or the data Accman would send to ALMA if a type-127 transaction were to be generated (as a kind of preview).

System Diagnostics

This would be a completely new page in Accman that can show all kinds of health-checks such as

  1. Current status of External Systems 
  2. Which transactions are queuing up
  3. Which back-end jobs are currently running
