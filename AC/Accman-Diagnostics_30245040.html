Introduction

Accman has a number of diagnostic features to try and make it easier to fix problems with accounts.

Account-specific diagnostic information can be found on the Diagnostic Information tab when viewing account details for an individual account. This shows statistics like when the person last used their ID card. It will also show what data Accman would send to ALMA if an Update ALMA transaction were to be executed (which can now be done via Page 2 on the Account Actions page). Additional features will be added to this tab as new uses for it are identified.

System-wide diagnostics can be accessed through the System Status icon on the home page

You will be presented with (currently) three new tabs:

  1. System Status - checks if various databases and external systems are working
  2. SAP/Banner Imports - displays a list of recent imports and whether they have been successful or not
  3. Transaction Status - shows important information about the state of transactions within the Accman database

System Status Tab

This performs a few different heath checks. It tests whether the following things are working

  1. Accman's Mail and Associate databases can be queried
  2. Active Directory can be queried (it tries to retrieve the AD account for the current user)
  3. Banner, SAP and Janus databases can be queried
  4. The current user's account can be retrieved from ALMA (an external service)
  5. Office365. Testing Office365 is a little harder so Accman check if there has been any successful communication with Office365 in the last 30 minutes. If not, this isn't necessarily a problem so a further check is performed to see if there have been any failures in the same timeframe. If there have been errors, an error is reported. 

The System Status tab re-runs each test at regular intervals. By default it will re-run them every 60 seconds. You can change the interval using the dropdown list near the top-right of the screen.

SAP/Banner Imports Tab

This screen displays a list of recent imports and if they were successful or not. This allows you to identify periods where there were problems with the data coming into Accman from it's two main feeds. 

Transaction Status Tab

At the top of this tab, the most-recently processed transaction is displayed. In the screenshot, you can see that a "Generate Password.txt File" transaction was processed at 12:20. This is updated every five seconds, so if you can see it changing, you can be confident that Accman transactions are currently being processed successfully. It's a quick way to see if the back end is up and running. Note however, just because this is changing doesn't mean everything is perfect. There could still be problems with individual transactions or external systems. 

The main part of the screen shows a breakdown of how many transactions are currently in the state selected in the dropdown list near the top right of the screen. By default this is set to 'Pending'. This allows you to see very quickly if there is a build-up of a certain kind of transaction, indicating a problem that may need investigating. If you click on the Details button a list of those transactions will be shown. Only the first 100 are displayed initially, but at the bottom of the list is a link that will bring the next 100 results. 

 

Each row shows the main details pertaining to the transaction, including the user it was connected to (where applicable). You can click on the username to go to the user's main details page. From there, you can view the same transaction on the transaction history tab. If the transaction failed for some reason, you can access the log file from here to see what went wrong.
