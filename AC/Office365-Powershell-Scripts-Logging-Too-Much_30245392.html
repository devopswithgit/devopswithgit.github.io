JIRA Issue(s) ARF-410 - Office365 Powershell Scripts Logging Too Much Resolved

Details

There are many gigabytes worth of temporary log files in C:\users\svc-prd-accman\appdata\local\microsoft\office365\powershell on the Accman Back End server (uos-p-acbe-01).

These can be deleted to free up space but they shouldn't be getting written to the disk in the first place. They are not log files that we explicitly write ourselves and we do not need them. They seem to be written by the Powershell framework itself, rather than by our own scripts.

There are many posts on the internet such as this one which suggest that it could be a result of less than perfect error handling in our Powershell Scripts (the Office365 ones running in Accman). However tracking down what is actually causing this logging will take some investigation. 

The purpose of this issue is therefore to investigate what could be generating these logs and then fix whatever Powershell scripts are responsible. In the meantime, we will have to occasionally delete the log files when they start filling up the disk.
