Accman : Notification when 'printed' accounts run out not working

Created by Martin Evans, last modified on Feb 16, 2016
SummaryNotification when 'printed' accounts run out not working
 
Issue TypeBug
PriorityHigh
Raised ByMartin Evans
Date Raised16/02/2016
SourceMartin Evans
JIRA Issue(s)

 

Details

  • When the account assignment process runs out of 'printed' accounts for a particular Dept/Level, we currently have no idea this has happened. It only comes to light when someone from the business or the students themselves report that they don't appear to have a username.
  • There used to be an email that came out to the Apps Support mailbox so we knew when it had happened so we could generate more. This doesn't happen now.
  • This needs reconstituting. We need a notification when there are, say, 100 left for a dept/level so that we can generate a new bunch before it runs out.
  • It can impact on some new students’ experiences if they are unlucky to be the ones to be assigned after it runs out as they don't receive registration emails that they should do or just discover they haven't been given a username or email.
  • SA are not happy with the situation as is and will be raising this as an issue.
