JIRA Issue(s)

 

Details

Every top levle SAP department codes (4 char) are mapped into AccMan Sap_dept table. These table is used by Sap_import transaction to map sap department to accman department. For some reason if none matched, accman department code set to 'ZZ'.

Historically HR Department sent most up-to-date sap department list to Andy Bourne. Who then populate into Accman. As per Matt Davies notes there used to be planning unit who manage this accman department codes. Which is no longer exists.

What I found is there are three Sap departments which are currently missing in AccMan.

 

  1. CAQE Quality & Enhancement Office - Created 1 March 2015

  2. CASS Student & Academic Services - Created 1 March 2015

  3. ZZSP NonUoS-Salford Professional Development

I believe these needs to populated in sap_dept table in Mail Database.

Impact

If we don't add these departments means staff self activation won't work. However their AD account will be activated and they will be able to use the university system as normal.

Fix

We will have to add or map dept in DEPT and SAP_DEPT table in Mail Database.
