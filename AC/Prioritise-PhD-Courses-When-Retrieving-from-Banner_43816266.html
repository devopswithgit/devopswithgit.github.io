JIRA Issue(s)

ARF-502 - Prioritise PhD Courses Open


Details

Unfortunately, where a student is registered on more than one course in Banner, only one course can be registered on their Active Directory contact record. In case 232766 the course code registered on student contact card is for the Postgrad Cert. course. For this reason he is not included when the "Postgraduate Research FT in SBS" Distribution Group is compiled, as this is built from a query on the course code in AD. The dynamic distribution group is rebuilt from the query on each use, which means that student's address cannot be manually added to the group. The only possible resolution is for the primary course registration to be altered to PHD/BM1/F on the contact record. This may in turn affect any mailings based on the Academic Practice course. In addition this code will be reset whenever AD is being updated from the Banner course information by ACCMAN. We need to persuade ACCMAN to pick up any PhD courses from Banner as priority for populating AD.
We get one or two cases every year or so with this issue.
