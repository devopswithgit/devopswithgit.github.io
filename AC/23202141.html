Accman : Transactions remains in 'creating' state

Created by Unknown User (ujs079), last modified on Oct 20, 2015
SummarySome newly created transactions remain in the 'creating' state which results in them not being executed.  
Issue TypeBug
PriorityLow
Raised ByUnknown User (ujs079)
Date Raised16/10/2015
SourceThis was originally raised by Matthew Davies. He noticed transaction 61 (Change Outlook Live [O365] Password) for several users was created with parameters but was never run because it was in the 'creating' state. From conversations with other service desk members it appears this is not specific to any one transaction.
 
JIRA Issue(s) ARF-276 - Transactions remain in 'creating' state Closed
LanDESKINC/171689

 

Details

Once a transaction is created (in the 'creating' state) parameters used by it are added. Whilst attempting to update the new transaction state to 'waiting' a TimeoutException is generated and the transaction state remains in the 'creating' state.

Transactions in the creating state are never executed. It is an interim state where parameters are being loaded. Once this has completed the state should be altered to 'waiting'.

The error occurrences on multiple transaction types intermittently. This makes it difficult to find the route cause. It may be due to a table lock when creating the initial transaction record or a slowdown in the performance of the database caused by additional load at the time of creating the transaction.
