Accman : Email for Life

Introduction

In a nutshell, Email4Life is a modification to Accman which means that a student who has graduated (and therefore left the university) can continue using their Salford email address for ever. 

Previously, upon leaving the university, a student's email account would be deleted when their account was disabled.

Process 

  1. Student graduates
  2. Accman reads the details from Banner and discovers that the student has gone
  3. Accman checks if the student left for a good reason (they graduated and are now an alumnus) or a bad reason (kicked off the course for not attending etc etc)
  4. If the reason is good, disable their Accman account but do not disable their Live@Edu email address. 
  5. Also, hide their address from the Live@Edu address list.

The first part of the process (deciding whether the student is an alumnus and should therefore keep their email address) takes place in the banner_import Perl scripts.

This is always the case with Accman, since to get scripts working at all on test, we always have to make lots of manual changes because things like configuration is not stored in one location - it's spread throughout the scripts. We will have to do the following:

  1. Copy all the Perl scripts from test to a local development PC e.g. C:\AccmanCompare\FromTest
  2. Copy all the production scripts to a similar folder e.g. C:\AccmanCompare\FromProduction
  3. Use something like TotalCommander to compare all the files and see what's changed
  4. Identify which changes have been made just to get them to work at all in test (database connection strings etc)
  5. Identify which changes are specific to Email4Life and DepartmentChange - these are the changes we want to copy into production
  6. Copy the changes over the the production scripts
  7. Put the modified changes onto the production server
  8. Restart all the Perl scripts
