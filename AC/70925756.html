Accman : Error: Unable to retrieve your university email address

Created by Mo Mumtaz, last modified on Jul 08, 2020

Symptoms:

OLR user has reached stage 2 and is given error message; ‘Unable to retrieve your university email address’

This is caused as the AD account and office 365 accounts have not yet been generated. Likely an issue with data migration between Banner and Accman.

Follow the workflow below to find the probable cause and solution.



SQL Select for PBANNO


SELECT DISTINCT

SPRIDEN_PIDM, SPRIDEN_ID, SPRIDEN_LAST_NAME, SPRIDEN_FIRST_NAME, SPRIDEN_MI,

SPBPERS_BIRTH_DATE, SPBPERS_SEX, SPBPERS_NAME_PREFIX, gm.GOREMAL_EMAIL_ADDRESS, v_lat.sprtele_intl_access as MOBILE_NUMBER

,sgb.SGBSTDN_CAMP_CODE as CAMPUS_CODE

FROM SPRIDEN

JOIN sgbstdn sgb ON spriden_pidm = sgb.sgbstdn_pidm

LEFT OUTER JOIN SPBPERS ON SPRIDEN_PIDM = SPBPERS_PIDM

LEFT JOIN

( SELECT sprtele_pidm, sprtele_intl_access

FROM sprtele s1

WHERE sprtele_tele_code = 'MOB'

AND sprtele_seqno = (SELECT max(sprtele_seqno)

FROM sprtele s2

WHERE sprtele_tele_code = 'MOB'

AND sprtele_primary_ind = 'Y'

AND s2.sprtele_pidm = s1.sprtele_pidm

AND sprtele_status_ind is null)

UNION ALL

SELECT sprtele_pidm, sprtele_intl_access

FROM sprtele s1

WHERE sprtele_tele_code = 'MOB'

AND sprtele_seqno = (SELECT max(sprtele_seqno)

FROM sprtele s2

WHERE sprtele_tele_code = 'MOB'

AND sprtele_status_ind is null

AND s2.sprtele_pidm = s1.sprtele_pidm

AND 0 = (SELECT COUNT(*) from sprtele s3 where sprtele_tele_code = 'MOB' and sprtele_primary_ind = 'Y' AND s3.sprtele_pidm = s2.sprtele_pidm))) v_lat

ON SPRIDEN_PIDM = v_lat.SPRTELE_PIDM

LEFT JOIN v_sal_shrdgmr_max_seq_no v ON spriden_pidm = v.shrdgmr_pidm

AND sgb.SGBSTDN_PROGRAM_1 = v.shrdgmr_program

LEFT JOIN ( SELECT goremal_pidm,goremal_email_address

FROM goremal

WHERE GOREMAL_PREFERRED_IND = 'Y' AND GOREMAL_EMAL_CODE = 2 AND GOREMAL_STATUS_IND = 'A'

UNION ALL

SELECT goremal_pidm,goremal_email_address

FROM goremal

WHERE GOREMAL_EMAL_CODE = 2 AND GOREMAL_STATUS_IND = 'A'

AND 0 = ( SELECT count (*) from goremal where GOREMAL_PREFERRED_IND = 'Y' AND GOREMAL_EMAL_CODE = 2 AND GOREMAL_STATUS_IND = 'A') )

GM

ON spriden_pidm = gm.goremal_pidm

WHERE SPRIDEN_CHANGE_IND IS NULL

AND SPRIDEN_PIDM = [PIDM];

Mandatory fields

The above SQL query should return at minimum;

-SPRIDEN_PIDM

-SPRIDEN_ID

-SPRIDEN_LAST_NAME

-SPRIDEN_FIRST_NAME

-SPBPERS_BIRTH_DATE

-CAMPUS_CODE

If these fields are not returned then it may be the reason for inccorrect data migration to Accman due to missing data fields.
